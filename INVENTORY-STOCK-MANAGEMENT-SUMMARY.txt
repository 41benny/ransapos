5. USER FLOW & SCENARIOS
=================================================================

SCENARIO 1: MONITORING STOK
1. Admin buka menu "Inventory & Stok"
2. Lihat dashboard dengan statistics cards
3. Filter by outlet atau category
4. Identifikasi produk dengan stok minimum/habis
5. Klik "Kartu Stok" untuk lihat detail history

SCENARIO 2: STOCK ADJUSTMENT (OPNAME)
1. Admin klik "Stock Adjustment"
2. Pilih outlet dan produk
3. Sistem show current stock otomatis
4. Input actual stock hasil opname
5. Sistem hitung dan tampilkan selisih
6. Input notes/alasan adjustment
7. Submit → Stok terupdate & mutasi tercatat

SCENARIO 3: TRANSFER STOK ANTAR OUTLET
A. CREATE TRANSFER
   1. Admin klik "Transfer Stok"
   2. Pilih outlet pengirim dan penerima
   3. Set tanggal transfer
   4. Add products yang akan ditransfer
   5. Sistem show available stock di outlet pengirim
   6. Submit → Transfer created (status: pending)

B. SEND TRANSFER
   1. Dari detail transfer, klik "Kirim Transfer"
   2. Konfirmasi
   3. Sistem:
      - Reduce stock di outlet pengirim
      - Create mutation (transfer_out)
      - Update status → in_transit

C. RECEIVE TRANSFER
   1. Outlet penerima buka detail transfer
   2. Klik "Terima Barang"
   3. Input actual quantity received
   4. Jika ada selisih, sistem warning
   5. Konfirmasi
   6. Sistem:
      - Add stock di outlet penerima
      - Create mutation (transfer_in)
      - Jika ada selisih, create adjustment
      - Update status → received

D. CANCEL TRANSFER
   1. Dari detail transfer (jika pending/in_transit)
   2. Klik "Batalkan"
   3. Input alasan pembatalan
   4. Konfirmasi
   5. Sistem:
      - Jika sudah sent, return stock ke pengirim
      - Create adjustment mutation
      - Update status → cancelled
