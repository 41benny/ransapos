## 1. TUJUAN TUGAS KETUJUH

1. Menyediakan **modul Pembelian Barang (Purchase)**:
   - Dijalankan oleh Admin / Manager (bukan kasir).
   - Menyimpan data pembelian dari supplier.
   - Bisa mencatat beberapa item per transaksi.

2. Saat pembelian **dikonfirmasi / diterima**:
   - Menambah stok produk di outlet terkait.
   - Menambahkan catatan mutasi stok (`type: in`).

3. Menyediakan tampilan list dan detail pembelian di Back Office:
   - List semua purchase dengan status.
   - Detail item pembelian dan total nilai.
   - Filter by tanggal, outlet, supplier, status.

4. Membuat service layer:
   - `PurchaseService` untuk handle logic & integrasi ke `StockService`.

## 2. STRUKTUR DATABASE

Struktur tabel `purchases` dan `purchase_items` **sudah ada** dari Tugas 1.  
Tambahkan/cek agar field minimal seperti berikut:

### Table: `purchases`
| Kolom | Tipe | Keterangan |
|--------|------|------------|
| id | bigIncrements | Primary Key |
| purchase_number | string | Nomor unik (contoh: `PO-OUT001-20251108-001`) |
| outlet_id | fk → outlets | Outlet tujuan |
| supplier_id | fk → suppliers | Supplier |
| user_id | fk → users | Dibuat oleh siapa |
| purchase_date | date | Tanggal dokumen |
| status | enum('draft','received','cancelled') | Status pembelian |
| subtotal | decimal | Total harga item sebelum pajak |
| tax | decimal | Opsional (default 0) |
| total_amount | decimal | Total akhir |
| notes | text nullable | Catatan tambahan |
| created_at / updated_at | timestamps |

### Table: `purchase_items`
| Kolom | Tipe | Keterangan |
|--------|------|------------|
| id | bigIncrements | Primary Key |
| purchase_id | fk → purchases | Relasi ke header |
| product_id | fk → products | Barang yang dibeli |
| quantity | decimal | Jumlah unit |
| price | decimal | Harga per unit |
| total | decimal | quantity × price |
| notes | text nullable | Opsional |
| created_at / updated_at | timestamps |

Tambahkan migration baru jika kolom tersebut belum lengkap.  
Pastikan semua FK & index sudah ada.
