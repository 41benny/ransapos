# EXPENSE MANAGEMENT MODULE - IMPLEMENTATION SUMMARY

## Overview
Complete Expense & Cost Management system has been implemented for PT Dua Putra Sekincau's Back Office application. This module provides comprehensive expense tracking with approval workflow, category management, cash account integration, and detailed reporting capabilities.

## Implementation Date
November 20, 2025

## Module Features

### 1. Expense Category Management
**Hierarchical Category System:**
- Parent and child category structure
- Each category can be linked to COA (Chart of Accounts) for accounting integration
- Active/inactive status management
- Custom ordering for display
- Category-wise expense tracking

**Key Features:**
- Create/edit/delete expense categories
- Parent-child relationship support
- COA account linking
- Description and notes
- Display order management

### 2. Expense Management
**Expense Tracking:**
- Complete expense record with auto-numbering (Format: EXP-OUTLET-YYYYMMDD-XXX)
- Expense date, amount, description
- Payment method tracking (cash, transfer, credit card, debit card, other)
- Reference number / invoice number
- File attachment support (receipt/proof - JPG, PNG, PDF max 2MB)
- Outlet-based expense tracking
- Category classification

**Approval Workflow:**
- 4-stage status: Pending → Approved/Rejected → Paid
- Approval/rejection with notes
- Approver tracking
- Approval timestamp

**Payment Processing:**
- Cash/bank account integration
- Automatic cash transaction creation on payment
- Balance deduction from cash accounts
- Payment status tracking

### 3. Expense Reports & Analytics
**Statistical Dashboard:**
- Total expenses summary
- Pending count and amount
- Approved count
- Paid count and amount
- Rejected count

**Visual Analytics:**
- Expense by Category (Doughnut Chart)
- Expense Trend by Month (Line Chart - 12 months)
- Interactive Chart.js visualizations

**Report Features:**
- Filter by outlet
- Filter by date range
- Expense breakdown by category
- Transaction count per category
- Percentage distribution
- Detailed table with totals

## Technical Implementation

### Database Tables

#### 1. expense_categories
```
id                  : Primary key
name                : Category name (varchar 100)
code                : Unique code (varchar 20)
parent_id           : Self-referencing FK (nullable)
coa_account_id      : FK to coa_accounts (nullable)
description         : Text description (nullable)
is_active           : Boolean (default true)
order               : Integer for sorting (default 0)
timestamps
```

Indexes:
- code
- is_active
- parent_id

#### 2. expenses
```
id                  : Primary key
expense_number      : Unique expense number (varchar 50)
outlet_id           : FK to outlets
expense_category_id : FK to expense_categories
expense_date        : Date
amount              : Decimal(15,2)
payment_method      : Varchar(50)
cash_account_id     : FK to cash_accounts (nullable)
reference_no        : Reference/invoice number (varchar 100, nullable)
description         : Text
attachment_path     : File path (nullable)
status              : Enum (pending, approved, rejected, paid) default pending
created_by          : FK to users (nullable)
approved_at         : Timestamp (nullable)
approved_by         : FK to users (nullable)
approval_notes      : Text (nullable)
timestamps
```

Indexes:
- expense_date
- status
- outlet_id + expense_date (composite)
- expense_category_id + expense_date (composite)

### Models

**Location:** `app/Models/ExpenseCategory.php`

**Relationships:**
- belongsTo: parent (ExpenseCategory), coaAccount (CoaAccount)
- hasMany: children (ExpenseCategory), expenses (Expense)

**Scopes:**
- active() - Filter active categories
- parents() - Filter parent categories only

**Attributes:**
- full_name - Returns "Parent > Child" for sub-categories

**Helper Methods:**
- isParent() - Check if category is parent
- hasChildren() - Check if has sub-categories

#### Expense Model
**Location:** `app/Models/Expense.php`

**Relationships:**
- belongsTo: outlet, category (ExpenseCategory), cashAccount, creator (User), approver (User)

**Scopes:**
- byOutlet($outletId)
- byCategory($categoryId)
- dateRange($startDate, $endDate)
- byStatus($status)

**Attributes:**
- status_badge - Returns color for status badge
- status_label - Returns human-readable status

**Helper Methods:**
- isPending(), isApproved(), isRejected(), isPaid()
- canBeApproved(), canBeRejected(), canBePaid()

### Service Layer

#### ExpenseService
**Location:** `app/Services/ExpenseService.php`

**Key Methods:**

1. **generateExpenseNumber($outletId, $date)**
   - Format: EXP-OUTLET-YYYYMMDD-XXX
   - Auto-increment daily sequence per outlet

2. **createExpense($data)**
   - Creates expense record
   - Handles file upload
   - Sets creator
   - Creates cash transaction if status is 'paid'

3. **updateExpense($expense, $data)**
   - Updates expense information
   - Handles file replacement
   - Only for pending expenses

4. **approveExpense($expense, $notes)**
   - Changes status to approved
   - Records approver and timestamp
   - Validates can be approved

5. **rejectExpense($expense, $reason)**
   - Changes status to rejected
   - Records rejection reason
   - Validates can be rejected

6. **payExpense($expense, $cashAccountId)**
   - Changes status to paid
   - Creates cash transaction (debit/expense type)
   - Reduces cash account balance
   - Validates can be paid

7. **deleteExpense($expense)**
   - Deletes attachment file
   - Reverses cash transaction if paid
   - Removes expense record

8. **getStatistics($outletId, $startDate, $endDate)**
   - Returns summary statistics
   - Total expenses, counts by status
   - Total paid and pending amounts

9. **getExpenseByCategory($outletId, $startDate, $endDate)**
   - Returns expense grouped by category
   - Total amount and count per category
   - Excludes rejected expenses

10. **getExpenseTrend($outletId, $months)**
    - Returns monthly expense trend
    - Last N months data
    - Total amount and count per month

## Workflow Diagram

### Expense Lifecycle
```
1. CREATE
   └─> Status: Pending
       ├─> APPROVE ──> Status: Approved
       │   └─> PAY ──> Status: Paid (Cash Transaction Created)
       └─> REJECT ──> Status: Rejected

2. EDIT (Only Pending)
3. DELETE (Only Pending, or Paid with cash reversal)
```

### Approval Workflow
```
User Creates Expense (Pending)
    │
    ├─> Manager/Admin Reviews
    │   ├─> APPROVE (with notes)
    │   │   └─> Finance Processes Payment
    │   │       └─> Mark as PAID
    │   │           └─> Cash Transaction Created
    │   │           └─> Cash Account Balance Reduced
    │   └─> REJECT (with reason)
    │       └─> End (No payment, no cash transaction)
    │
    └─> Or Direct Create as PAID (for already paid expenses)
        └─> Cash Transaction Created immediately
```

## Integration Points

### 1. Cash Account Integration
- When expense status = 'paid' and cash_account_id is set
- Creates CashTransaction with type = 'expense'
- Reduces cash account balance
- Links to expense via reference_type and reference_id
- On delete: Reverses transaction and restores balance

### 2. COA Integration
- Each expense category can link to COA account
- Enables accounting integration
- Provides mapping for financial reports
- Future: Auto-post journal entries

### 3. Outlet Integration
- All expenses tagged to specific outlet
- Enables outlet-wise expense tracking
- Multi-location expense analysis
- Cost center reporting

### 4. User Integration
- Tracks expense creator
- Tracks approver/rejector
- Approval timestamp
- Audit trail for accountability
