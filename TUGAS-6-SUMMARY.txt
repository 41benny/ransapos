10. FLOW AUTENTIKASI
================================================================================

Login Flow:
1. User akses / → redirect ke /login
2. User input email & password
3. System validasi kredensial
4. Cek is_active = true
5. Login berhasil → redirect berdasarkan role:
   - Admin/Manager → /admin/dashboard
   - Kasir → /pos/dashboard

Protected Routes:
1. User akses route protected tanpa login → redirect ke /login
2. User login tapi role tidak sesuai → 403 Forbidden

Logout Flow:
1. User klik tombol logout
2. Session dihapus
3. Redirect ke /login dengan pesan sukses
