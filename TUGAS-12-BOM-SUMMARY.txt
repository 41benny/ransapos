1. TUJUAN FITUR
================================================================================
Implementasi sistem BOM (resep produksi) untuk produk finished_good dengan 
skenario "made-to-order":
- Saat produk jadi dijual, TIDAK mengurangi stok produk jadi
- HANYA mengurangi stok bahan baku (raw materials) sesuai resep/BOM
- HPP (COGS) dihitung dari total biaya bahan yang digunakan
- Validasi stok bahan sebelum transaksi (tolak jika kurang)

4. LOGIKA BISNIS (SERVICE LAYER)
================================================================================

File: app/Services/SaleService.php

Perubahan method createSale():

FLOW PENGURANGAN STOK PER SALE ITEM:
┌─────────────────────────────────────────────────────────────────┐
│ 1. Load product dengan eager load bomHeader (jika aktif)       │
│ 2. Cek product_type:                                            │
│                                                                 │
│    A. raw_material:                                             │
│       → Kurangi stok produk langsung (perilaku lama)           │
│                                                                 │
│    B. finished_good:                                            │
│       • Jika punya BOM aktif:                                   │
│         - Validasi SEMUA stok komponen cukup (loop pertama)    │
│         - Jika kurang → throw Exception                        │
│         - Jika cukup → kurangi stok tiap komponen (loop kedua) │
│         - Qty konsumsi = detail.quantity × sale_item.quantity  │
│         - Catat stock_mutation per komponen (reference: sale)  │
│       • Jika TIDAK punya BOM aktif:                            │
│         - Fallback: kurangi stok produk jadi (perilaku lama)   │
│                                                                 │
│    C. service:                                                  │
│       → Tidak ada pengurangan stok                             │
│                                                                 │
│ 3. Semua dalam DB transaction (atomic)                          │
└─────────────────────────────────────────────────────────────────┘

Fitur Guard Stok:
- Sebelum konsumsi, cek getAvailableStock() per komponen
- Jika tidak cukup: throw Exception dengan pesan jelas
- Default: TOLAK transaksi (tidak boleh minus)

HPP/COGS:
- StockService::reduceSaleStock() sudah mencatat unit_cost & total_cost
- Cost diambil dari product.purchase_price
- Mutasi stok komponen otomatis tercatat untuk laporan Laba Rugi

5. CONTROLLER & API ENDPOINT
================================================================================

File: app/Http/Controllers/Admin/BomController.php

Resource Routes (JSON API):
┌────────────────────────────────────────────────────────────────┐
│ GET    /admin/boms           → index (list semua BOM)         │
│ POST   /admin/boms           → store (buat BOM baru)          │
│ GET    /admin/boms/{bom}     → show (detail BOM)              │
│ PUT    /admin/boms/{bom}     → update (edit BOM)              │
│ DELETE /admin/boms/{bom}     → destroy (hapus BOM)            │
└────────────────────────────────────────────────────────────────┘

Validasi Store:
- product_id wajib, harus finished_good (bukan raw_material/service)
- components array min 1 item
- component_product_id harus exist di products
- quantity min 0.0001
- Tidak boleh loop (component = product itu sendiri)

Validasi Update:
- Sama seperti store
- Strategi sederhana: hapus semua detail lama, insert baru

7. TESTING
================================================================================

File: tests/Feature/BomConsumptionTest.php

Test Case 1: it_consumes_raw_materials_instead_of_finished_good_stock_when_bom_active
Skenario:
- Produk: Nasi Goreng (finished_good)
- BOM: 0.2 kg Beras + 1 pcs Telur per porsi
- Stok awal: Beras 10 kg, Telur 50 pcs, Nasi Goreng 5 plate
- Jual: 3 porsi Nasi Goreng

Hasil:
✓ Stok Nasi Goreng tetap 5 (tidak berkurang)
✓ Stok Beras berkurang 0.6 kg (10 → 9.4)
✓ Stok Telur berkurang 3 pcs (50 → 47)
✓ 2 stock_mutations tercatat (untuk Beras & Telur)

Test Case 2: it_rejects_sale_when_component_stock_insufficient
Skenario:
- Produk: Sup (finished_good)
- BOM: 1 gram Garam per bowl
- Stok awal: Garam 0.5 gram (KURANG!)
- Jual: 1 bowl Sup

Hasil:
✓ Exception thrown dengan pesan "Stok bahan Garam tidak mencukupi"
✓ Transaksi dibatalkan (rollback)

Status Test: 2 tests, 5 assertions → ALL PASS ✓
