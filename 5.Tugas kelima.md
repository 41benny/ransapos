## 2. STRUKTUR ROUTE

Tambahkan routes baru di **area admin**.

### 2.1. Route Group Reports

Di `routes/web.php` (group `admin`), tambahkan:

- `GET /admin/reports/sales`           → daftar laporan penjualan (summary).
- `GET /admin/reports/sales-products`  → laporan penjualan per produk.
- `GET /admin/reports/shifts`          → laporan shift kasir (ringkasan).
- `GET /admin/reports/shifts/{id}`     → detail satu shift.

Nama route yang disarankan:

- `admin.reports.sales.index`
- `admin.reports.sales.products`
- `admin.reports.shifts.index`
- `admin.reports.shifts.show`

## 3. CONTROLLER – ADMIN REPORTS

Buat folder & controller:

- Folder: `app/Http/Controllers/Admin/Reports/`
- File:
  - `SalesReportController.php`
  - `ShiftReportController.php`

### 3.1. SalesReportController

Methods:

1. `index(Request $request)`
   - Input filter:
     - `date_from`, `date_to`
     - `outlet_id`
     - `cashier_id`
     - `payment_method_id` (opsional)
   - Query:
     - Ambil data dari `sales` + `payments` + `users` + `outlets`.
     - Filter berdasarkan field di atas.
   - Output ke view:
     - `summary`:
       - total_transactions
       - total_amount
       - avg_per_transaction
       - total_cash
       - total_non_cash (dari `payments` + `payment_methods`)
     - `details`:
       - list transaksi (tanggal, invoice_number, outlet, kasir, total, metode pembayaran utama).

2. `products(Request $request)`
   - Input filter sama (periode, outlet, kasir).
   - Query:
     - Dari `sale_items` gabung `sales` dan `products`.
   - Group by `product_id`:
     - `product_name`, `sku`
     - `total_qty`
     - `total_amount`
   - Output ke view sebagai tabel "Penjualan per Produk".

### 3.2. ShiftReportController

Methods:

1. `index(Request $request)`
   - Filter:
     - date range (berdasarkan `opened_at` / `closed_at`)
     - `outlet_id`
     - `cashier_id`
     - `status` (open/closed)
   - Query:
     - Dari `cash_sessions` + relasi `outlet`, `user`.
   - Output:
     - List shift dengan:
       - session_number
       - outlet
       - kasir
       - opened_at, closed_at
       - opening_balance
       - total_sales
       - expected_balance
       - actual_balance
       - difference
       - status

2. `show(CashSession $cashSession)`
   - Ambil:
     - Data session (`cash_sessions`).
     - Semua `sales` yang terkait session ini (jika ada relasi `cash_session_id` di `sales`; jika belum ada, gunakan cara yang dipakai di SaleService/CashSessionService untuk korelasinya).
     - Dari `sales` → ambil `payments`, `sale_items`.
   - Hitung:
     - Total transaksi.
     - Total sales.
     - Breakdown by payment method.
     - Top produk di shift ini (qty & omzet).
   - Kirim ke view detail.

> Jika di struktur saat ini `sales` belum punya `cash_session_id`, gunakan pendekatan yang paling konsisten dengan implementasi Tugas 3–4 (misalnya: ada relasi sudah dibuat; jangan ubah struktur tabel besar tanpa perlu).

## 4. VIEW – LAPORAN PENJUALAN

Buat folder view:

- `resources/views/admin/reports/`

File:

1. `sales/index.blade.php`
2. `sales/products.blade.php`
3. `shifts/index.blade.php`
4. `shifts/show.blade.php`

### 4.1. sales/index.blade.php

Isi:

- **Filter form** di atas (inline / grid):
  - Date from, date to (default: hari ini).
  - Outlet (dropdown).
  - Kasir (dropdown).
  - Metode pembayaran (dropdown “Semua / Cash / QRIS / dst”).
  - Tombol `Filter` & `Reset`.

- **Summary cards** (4 kartu):
  - Total Transaksi
  - Total Omzet
  - Rata-rata per Transaksi
  - Total Cash vs Non-Cash

- **Tabel detail transaksi**:
  - Kolom:
    - Tanggal & Jam
    - Invoice
    - Outlet
    - Kasir
    - Metode pembayaran utama
    - Total
  - Total baris terakhir: jumlah transaksi & total omzet.

- Print-friendly:
  - Tambahkan CSS sederhana `@media print`:
    - Hilangkan tombol & filter.
    - Hanya tampilkan judul + summary + tabel.

### 4.2. sales/products.blade.php

Isi:

- Filter (periode, outlet, kasir).
- Tabel “Penjualan per Produk”:
  - Produk
  - SKU
  - Total Qty
  - Total Omzet
  - Rata-rata harga (total_amount / total_qty)
- Urutkan descending berdasarkan `total_amount` (produk terlaris di atas).
- Tampilkan grand total di bawah.

## 6. QUERY & PERFORMANSI

Saat menulis query:

- Gunakan **Eloquent dengan eager loading**:
  - `sales` dengan `outlet`, `user`, `payments`.
  - `sale_items` dengan `product`.
  - `cash_sessions` dengan `outlet`, `user`.

- Untuk agregasi:
  - Boleh gunakan `selectRaw` + `groupBy` untuk laporan per produk.
  - Hati-hati dengan double-count jika join ke `payments`.

- Batasi:
  - Jika periode > 31 hari, boleh tampilkan warning bahwa periode cukup besar.
