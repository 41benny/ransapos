3. SERVICES - BUSINESS LOGIC
================================================================================

✅ File: app/Services/CashAccountService.php

Methods yang Dibuat:

1. createAccount(array $data): CashAccount
   - Buat akun kas/bank baru
   - Set opening_balance = current_balance
   - Validasi & return dengan relasi

2. updateAccount(CashAccount $account, array $data): CashAccount
   - Update akun kas/bank
   - TIDAK bisa ubah opening_balance & current_balance via update
   - Return updated account

3. generateTransactionNumber(string $accountCode): string
   - Format: CT-{account_code}-{date}-{seq}
   - Contoh: CT-KAS001-20251107-001
   - Auto increment per hari per akun

4. recordTransaction(array $data): CashTransaction
   - Catat transaksi kas/bank (masuk/keluar)
   - Generate transaction number
   - Hitung balance_before & balance_after
   - Validasi saldo tidak boleh negatif
   - Update current_balance di akun
   - Return transaction dengan relasi

5. recordPurchasePayment(Purchase $purchase, array $data): CashTransaction
   - Catat pembayaran purchase (kas keluar)
   - Validasi purchase harus sudah received
   - Validasi jumlah tidak melebihi sisa tagihan
   - Create transaction dengan reference ke purchase
   - Update payment_status di purchase (pending/partial/paid)
   - Return transaction

6. getTransactions(array $filters = []): LengthAwarePaginator
   - Get list transaksi dengan filter
   - Filter by: cash_account_id, type, date_from, date_to, reference_type
   - Paginated (20 per page)
   - Eager load: cashAccount, creator

7. getMutationReport(int $cashAccountId, ?string $dateFrom, ?string $dateTo): array
   - Get laporan mutasi kas untuk 1 akun
   - Default: bulan ini
   - Return: account, transactions, date range, summary (beginning_balance,
     total_in, total_out, ending_balance)

8. getAccountsSummary(): array
   - Get ringkasan semua akun
   - Return: accounts, total_cash, total_bank, total_balance

10. FLOW BISNIS LENGKAP
================================================================================

✅ Flow 1: Kelola Akun Kas/Bank

Buat Akun Baru:
   1. Admin buka menu "Kas & Bank"
   2. Klik "Tambah Akun Baru"
   3. Isi form: nama, kode, jenis, saldo awal
   4. Submit → Akun dibuat dengan current_balance = opening_balance
   5. Redirect ke detail akun

Edit Akun:
   1. Dari list, klik Edit
   2. Ubah nama, kode, atau notes
   3. Submit → Akun diupdate
   4. SALDO TIDAK BISA DIUBAH (hanya via transaksi)

Hapus Akun:
   1. Dari edit page, klik "Hapus Akun"
   2. Konfirmasi
   3. Validasi: hanya bisa hapus jika belum ada transaksi
   4. Submit → Akun dihapus

✅ Flow 2: Catat Transaksi Kas/Bank Manual

Transaksi Masuk (Cash In):
   1. Admin klik "Catat Transaksi Baru"
   2. Pilih akun kas/bank
   3. Pilih jenis: "Kas Masuk"
   4. Input tanggal, jumlah, deskripsi
   5. Submit → System:
      - Generate transaction number
      - balance_before = current_balance akun
      - balance_after = balance_before + amount
      - Update current_balance akun
      - Save transaction
   6. Saldo akun bertambah

Transaksi Keluar (Cash Out):
   1. Admin klik "Catat Transaksi Baru"
   2. Pilih akun kas/bank
   3. Pilih jenis: "Kas Keluar"
   4. Input tanggal, jumlah, deskripsi
   5. Submit → System:
      - Validasi: balance_after tidak boleh < 0
      - Generate transaction number
      - balance_before = current_balance akun
      - balance_after = balance_before - amount
      - Update current_balance akun
      - Save transaction
   6. Saldo akun berkurang

✅ Flow 3: Pembayaran Purchase (Integration)

Receive Purchase:
   1. Admin receive purchase (status: draft → received)
   2. Stok bertambah via StockService
   3. payment_status masih 'pending' (belum bayar)

Catat Pembayaran Purchase:
   1. Dari detail purchase, klik "Catat Pembayaran"
   2. Validasi: purchase harus sudah received
   3. Form payment muncul:
      - Total purchase: Rp 10.000.000
      - Sudah dibayar: Rp 0
      - Sisa tagihan: Rp 10.000.000
   4. Pilih akun kas/bank (misal: Bank BCA)
   5. Input jumlah: Rp 5.000.000 (partial payment)
   6. Input tanggal & notes
   7. Submit → System:
      - Validasi: amount <= remaining
      - Call recordPurchasePayment()
      - Create cash transaction:
        * type: out
        * reference_type: purchase
        * reference_id: purchase->id
        * description: "Pembayaran Purchase #{purchase_number}"
      - Update saldo Bank BCA (berkurang Rp 5.000.000)
      - Update payment_status purchase: 'partial'
   8. Redirect ke detail purchase

Pembayaran Kedua (Pelunasan):
   1. Dari detail purchase, klik "Catat Pembayaran" lagi
   2. Sisa tagihan sekarang: Rp 5.000.000
   3. Input jumlah: Rp 5.000.000
   4. Submit → System:
      - Create cash transaction (kas keluar)
      - Update saldo akun
      - Update payment_status purchase: 'paid' (lunas!)
   5. Form pembayaran hilang, muncul alert "Purchase Sudah Lunas"

✅ Flow 4: Laporan Mutasi Kas

Lihat Laporan:
   1. Admin buka detail akun atau klik "Mutasi" dari list
   2. Default: mutasi bulan ini
   3. Ubah date range jika perlu
   4. Klik "Tampilkan"
   5. Laporan muncul:
      - Summary: Saldo Awal, Total Masuk, Total Keluar, Saldo Akhir
      - Tabel rincian per transaksi
      - Footer total
   6. Klik "Print" untuk print laporan
