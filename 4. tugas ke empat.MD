## 1. TUJUAN TUGAS KEEMPAT

1. Menyediakan fitur **OPEN SHIFT** kasir:
   - Kasir memilih outlet.
   - Mengisi saldo awal kas (opening balance).
   - Sistem membuat `cash_sessions` baru dengan status `open`.

2. Menyediakan fitur **CLOSE SHIFT** kasir:
   - Menampilkan ringkasan penjualan sesi tersebut.
   - Kasir input kas fisik aktual di laci.
   - Sistem hitung selisih (short/over).
   - Update `cash_sessions` → status `closed`, simpan `closed_at`, `actual_balance`, `difference`, `notes`.

3. **Membatasi transaksi POS**:
   - Kasir **tidak bisa membuat transaksi** kalau tidak ada `cash_session` dengan status `open` untuk user + outlet tersebut.

4. Membuat halaman **riwayat cash session** di Back Office:
   - Admin bisa lihat list shift per kasir/per outlet.
   - Bisa filter minimal by tanggal & status.

Gunakan Bahasa Indonesia untuk komentar & pesan error.

## 2. STRUKTUR DATA CASH SESSIONS

### 2.1. Pastikan Struktur Tabel

Cek tabel `cash_sessions`.  
Kalau sudah punya field mirip ini, gunakan & sesuaikan. Kalau belum lengkap, tambahkan field berikut melalui migration baru (bukan edit migration lama):

Field minimal yang diharapkan di `cash_sessions`:

- `id`
- `code` (string) → kode shift, contoh: `CS-OUT001-20251107-001`
- `outlet_id` (fk → outlets)
- `user_id` (fk → users) → kasir yang menjalankan shift
- `opened_at` (datetime)
- `closed_at` (nullable datetime)
- `opening_balance` (decimal, default 0)
- `total_sales` (decimal, default 0)
- `total_cash` (decimal, default 0)
- `total_non_cash` (decimal, default 0)
- `expected_balance` (decimal, default 0) → opening + total_cash
- `actual_balance` (nullable decimal) → kas fisik saat tutup shift
- `difference` (nullable decimal) → actual - expected (bisa negatif)
- `status` (enum: open/closed)
- `notes` (nullable text)

Buat migration baru jika perlu, misalnya:

- `database/migrations/xxxx_xx_xx_xxxxxx_update_cash_sessions_add_closing_fields.php`

Pastikan model `CashSession` di-update agar fillable & casts-nya sesuai.

## 3. ROUTING & CONTROLLER – POS (KASIR)

### 3.1. Route POS untuk Cash Session

Di `routes/web.php`, dalam group `pos`, tambahkan routes:

- `GET /pos/sessions/open` → form buka shift
- `POST /pos/sessions` → proses buka shift
- `GET /pos/sessions/close` → form tutup shift (untuk session aktif user tersebut)
- `POST /pos/sessions/{cashSession}/close` → proses tutup shift

Nama route (disarankan):

- `pos.sessions.open`
- `pos.sessions.store`
- `pos.sessions.close`
- `pos.sessions.close.store` (atau `pos.sessions.update`)

### 3.2. POS\CashSessionController

Buat controller baru:

- `app/Http/Controllers/POS/CashSessionController.php`

Method minimal:

- `open()` → tampilkan form buka shift.
- `store(OpenCashSessionRequest $request)` → simpan session baru.
- `close()` → tampilkan ringkasan session aktif + form input kas aktual + notes.
- `closeStore(CloseCashSessionRequest $request, CashSession $cashSession)` → proses close.

## 4. FORM REQUEST – VALIDASI CASH SESSION

### 4.1. OpenCashSessionRequest

File:

- `app/Http/Requests/POS/OpenCashSessionRequest.php`

Validasi:

- `outlet_id` → required, exists:outlets,id
- `opening_balance` → required, numeric, min:0

Logic tambahan:

- Pastikan TIDAK ada cash_session `status = open` untuk kombinasi:
  - `user_id = current user` (atau sementara hardcode user id kalau auth belum aktif)
  - `outlet_id = outlet_id yang dipilih`

Kalau ada yang masih open → gagal dengan pesan:
> “Masih ada shift kasir yang belum ditutup untuk outlet ini.”

### 4.2. CloseCashSessionRequest

File:

- `app/Http/Requests/POS/CloseCashSessionRequest.php`

Validasi:

- `actual_balance` → required, numeric, min:0
- `notes` → nullable, string, max:500
