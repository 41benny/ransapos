7. ALUR TRANSAKSI LENGKAP
================================================================================

USER FLOW:
----------
1. User buka /pos/sales
2. Lihat grid produk (8 products)
3. Klik produk → Masuk cart
4. Adjust quantity di cart (+/-)
5. Pilih payment method (dropdown)
6. Klik "Bayar"
7. Lihat alert konfirmasi
8. Proses berlangsung...
9. Lihat alert sukses + invoice number
10. Cart auto reset

BACKEND FLOW:
-------------
1. Browser POST ke /pos/sales
2. StoreSaleRequest validasi
3. SaleController→store()
4. SaleService→createSale():
   
   a) Generate invoice: INV-001-20251107-0001
   
   b) Hitung total:
      - Subtotal = Σ(qty × price - discount per item)
      - Discount = subtotal × discount_value (jika percentage)
      - Total = subtotal - discount + tax
   
   c) Simpan sale:
      INSERT INTO sales (invoice_number, outlet_id, ...)
   
   d) Loop items:
      FOR EACH item IN cart:
         - Ambil product data
         - Simpan sale_item:
           INSERT INTO sale_items (sale_id, product_id, ...)
         
         - StockService→reduceSaleStock():
           * Ambil/buat stock record
           * Validasi: stock.qty >= item.qty ?
           * Update: stock.qty -= item.qty
           * Catat mutation:
             INSERT INTO stock_mutations (
               type: out,
               quantity: -item.qty,
               stock_before: X,
               stock_after: X-item.qty,
               reference_type: sale,
               reference_id: sale.id
             )
   
   e) Simpan payment:
      INSERT INTO payments (sale_id, payment_method_id, ...)
   
   f) Update cash_session:
      - total_sales += total_amount
      - total_cash atau total_non_cash += total_amount
      - expected_balance = opening + total_cash

5. DB Commit (jika sukses)
6. Return JSON response
7. Browser tampilkan alert
8. Reset cart
