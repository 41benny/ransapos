3. PURCHASE SERVICE (BUSINESS LOGIC)
================================================================================

✅ File: app/Services/PurchaseService.php

Methods yang Dibuat:

1. createPurchase(array $data): Purchase
   - Buat purchase baru dengan status 'draft'
   - Generate purchase number otomatis
   - Hitung total dari items
   - Simpan header + items
   - Return purchase dengan relasi

2. updatePurchase(Purchase $purchase, array $data): Purchase
   - Hanya bisa update jika status draft
   - Hapus items lama, buat items baru
   - Recalculate total
   - Return updated purchase

3. receivePurchase(Purchase $purchase): Purchase
   - Validasi harus status draft
   - Ubah status jadi 'received'
   - Set received_at & received_by
   - TAMBAH STOK via StockService
   - Catat stock mutation (type: in)
   - Return purchase dengan relasi lengkap

4. cancelPurchase(Purchase $purchase, string $reason): Purchase
   - Tidak bisa cancel jika sudah received
   - Ubah status jadi 'cancelled'
   - Tambah reason ke notes
   - Return purchase

5. generatePurchaseNumber(int $outletId): string
   - Format: PO-{outlet}-{date}-{sequence}
   - Contoh: PO-001-20251108-001
   - Auto increment per hari per outlet

6. getPurchases(array $filters): LengthAwarePaginator
   - Filter by outlet_id
   - Filter by supplier_id
   - Filter by status
   - Filter by date range (date_from, date_to)
   - Return paginated (15 per page)
   - Eager load: outlet, supplier, creator, items

================================================================================
4. STOCK SERVICE UPDATE
================================================================================

✅ File: app/Services/StockService.php

Method Existing yang Digunakan:

addPurchaseStock(int $productId, int $outletId, float $quantity, int $purchaseId, ?int $userId): void
   - Tambah stok produk di outlet
   - Catat stock mutation dengan type 'in'
   - Reference type: 'purchase'
   - Reference id: purchase_id

Flow saat Receive Purchase:
1. PurchaseService::receivePurchase()
2. Loop setiap item
3. Call StockService::addPurchaseStock() untuk setiap item
4. Stok bertambah + mutasi tercatat

Custom Messages:

6. PURCHASE CONTROLLER
================================================================================

✅ File: app/Http/Controllers/Admin/PurchaseController.php

Methods yang Dibuat:

1. index(Request $request)
   - Tampilkan list purchases dengan filter
   - Load outlets & suppliers untuk dropdown
   - Return view: admin.purchases.index

2. create()
   - Load outlets, suppliers, categories + products
   - Return view: admin.purchases.create

3. store(StorePurchaseRequest $request)
   - Validasi via Form Request
   - Call PurchaseService::createPurchase()
   - Redirect ke show dengan success message

4. show(Purchase $purchase)
   - Load relasi: items.product, outlet, supplier, creator, receiver
   - Return view: admin.purchases.show

5. edit(Purchase $purchase)
   - Validasi harus status draft
   - Load data untuk form
   - Return view: admin.purchases.edit

6. update(UpdatePurchaseRequest $request, Purchase $purchase)
   - Validasi via Form Request
   - Call PurchaseService::updatePurchase()
   - Redirect ke show dengan success message

7. receive(Purchase $purchase)
   - Call PurchaseService::receivePurchase()
   - Stok otomatis bertambah
   - Redirect ke show dengan success message

8. cancel(Request $request, Purchase $purchase)
   - Ambil reason dari request
   - Call PurchaseService::cancelPurchase()
   - Redirect ke show dengan success message

9. destroy(Purchase $purchase)
   - Hanya bisa delete jika status draft
   - Hapus purchase + items (cascade)
   - Redirect ke index dengan success message

11. FLOW PEMBELIAN
================================================================================

✅ Create Purchase (Draft):
   1. Admin klik "Buat Pembelian Baru"
   2. Pilih outlet & supplier
   3. Input tanggal pembelian
   4. Tambah items (produk, qty, harga)
   5. Set tax & discount (optional)
   6. Save → Status: Draft
   7. Purchase number auto generate
   8. STOK BELUM BERTAMBAH

✅ Edit Purchase (Draft Only):
   1. Dari list, klik Edit (hanya jika draft)
   2. Ubah data yang diinginkan
   3. Save → masih status Draft
   4. STOK BELUM BERTAMBAH

✅ Receive Purchase (Draft → Received):
   1. Dari detail, klik tombol "Terima Barang"
   2. Konfirmasi
   3. System:
      - Ubah status jadi 'received'
      - Set received_at & received_by
      - Loop setiap item
      - Tambah stok via StockService
      - Catat stock mutation (type: in, reference: purchase)
   4. STOK SUDAH BERTAMBAH
   5. Tidak bisa diubah/dihapus lagi

✅ Cancel Purchase (Draft → Cancelled):
   1. Dari detail, klik tombol "Batalkan"
   2. Input alasan pembatalan
   3. Konfirmasi
   4. System:
      - Ubah status jadi 'cancelled'
      - Tambah reason ke notes
   5. STOK TIDAK BERUBAH
   6. Tidak bisa diubah/dihapus lagi

✅ Delete Purchase (Draft Only):
   1. Dari detail, klik Delete
   2. Konfirmasi
   3. System hapus purchase + items (cascade)
   4. STOK TIDAK BERUBAH
