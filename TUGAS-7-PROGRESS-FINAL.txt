3. PURCHASE SERVICE (BUSINESS LOGIC)
================================================================================

✅ File: app/Services/PurchaseService.php

METHODS:

1. createPurchase(array $data): Purchase
   ┌─────────────────────────────────────────────────────────────────────┐
   │ FUNGSI:                                                             │
   │ - Buat purchase baru dengan status 'draft'                         │
   │ - Generate purchase number otomatis                                │
   │ - Hitung total dari items                                          │
   │ - Simpan header + items dalam 1 transaksi database                 │
   │                                                                     │
   │ INPUT: array dengan key:                                           │
   │   - outlet_id, supplier_id, purchase_date                          │
   │   - items[] (array of: product_id, quantity, unit_price, discount) │
   │   - tax_amount, discount_amount (optional)                         │
   │   - notes (optional)                                               │
   │                                                                     │
   │ OUTPUT: Purchase object dengan relasi lengkap                      │
   │                                                                     │
   │ VALIDATION: Via StorePurchaseRequest                               │
   └─────────────────────────────────────────────────────────────────────┘

2. updatePurchase(Purchase $purchase, array $data): Purchase
   ┌─────────────────────────────────────────────────────────────────────┐
   │ FUNGSI:                                                             │
   │ - Update purchase (HANYA JIKA STATUS DRAFT)                        │
   │ - Hapus items lama, buat items baru                                │
   │ - Recalculate total                                                │
   │                                                                     │
   │ EXCEPTION: Jika status bukan draft                                 │
   └─────────────────────────────────────────────────────────────────────┘

3. receivePurchase(Purchase $purchase): Purchase
   ┌─────────────────────────────────────────────────────────────────────┐
   │ FUNGSI: **INI YANG PALING PENTING!**                               │
   │ - Validasi harus status draft                                      │
   │ - Update status → 'received'                                       │
   │ - Set received_at = now()                                          │
   │ - Set received_by = auth()->id()                                   │
   │ - Loop setiap item:                                                │
   │   * Call StockService::addPurchaseStock()                          │
   │   * STOK BERTAMBAH OTOMATIS                                        │
   │   * MUTASI STOK TERCATAT (type: in, reference: purchase)           │
   │                                                                     │
   │ RESULT: Stok produk di outlet bertambah sesuai qty items           │
   └─────────────────────────────────────────────────────────────────────┘

4. cancelPurchase(Purchase $purchase, string $reason): Purchase
   ┌─────────────────────────────────────────────────────────────────────┐
   │ FUNGSI:                                                             │
   │ - Batalkan purchase (TIDAK BISA JIKA SUDAH RECEIVED)               │
   │ - Update status → 'cancelled'                                      │
   │ - Tambah reason ke notes                                           │
   │                                                                     │
   │ CATATAN: Stok tidak berubah                                        │
   └─────────────────────────────────────────────────────────────────────┘

5. generatePurchaseNumber(int $outletId): string
   ┌─────────────────────────────────────────────────────────────────────┐
   │ FORMAT: PO-{outlet}-{date}-{sequence}                              │
   │ CONTOH: PO-001-20251108-001                                        │
   │                                                                     │
   │ LOGIC:                                                             │
   │ - Auto increment sequence per hari per outlet                      │
   │ - Cari purchase terakhir hari ini di outlet                        │
   │ - Extract nomor terakhir + 1                                       │
   └─────────────────────────────────────────────────────────────────────┘

6. getPurchases(array $filters): LengthAwarePaginator
   ┌─────────────────────────────────────────────────────────────────────┐
   │ FUNGSI: List purchases dengan filter & pagination                  │
   │                                                                     │
   │ FILTERS:                                                           │
   │ - outlet_id (exact match)                                          │
   │ - supplier_id (exact match)                                        │
   │ - status (exact match)                                             │
   │ - date_from (purchase_date >=)                                     │
   │ - date_to (purchase_date <=)                                       │
   │                                                                     │
   │ OUTPUT: 15 items per page, sorted by date DESC                     │
   └─────────────────────────────────────────────────────────────────────┘

6. PURCHASE CONTROLLER
================================================================================

✅ File: app/Http/Controllers/Admin/PurchaseController.php

METHODS & ENDPOINTS:

1. index(Request $request)
   URL: GET /admin/purchases
   FUNGSI: List purchases dengan filter
   VIEW: admin.purchases.index
   DATA: purchases (paginated), outlets, suppliers

2. create()
   URL: GET /admin/purchases/create
   FUNGSI: Form buat purchase baru
   VIEW: admin.purchases.create
   DATA: outlets, suppliers, categories (with products)

3. store(StorePurchaseRequest $request)
   URL: POST /admin/purchases
   FUNGSI: Simpan purchase baru
   REDIRECT: admin.purchases.show
   FLASH: success message dengan purchase_number

4. show(Purchase $purchase)
   URL: GET /admin/purchases/{id}
   FUNGSI: Detail purchase
   VIEW: admin.purchases.show
   DATA: purchase dengan relasi lengkap

5. edit(Purchase $purchase)
   URL: GET /admin/purchases/{id}/edit
   FUNGSI: Form edit (HANYA DRAFT)
   VALIDATION: Redirect jika bukan draft
   VIEW: admin.purchases.edit
   DATA: purchase, outlets, suppliers, categories

6. update(UpdatePurchaseRequest $request, Purchase $purchase)
   URL: PUT /admin/purchases/{id}
   FUNGSI: Update purchase (HANYA DRAFT)
   REDIRECT: admin.purchases.show
   FLASH: success message

7. receive(Purchase $purchase)
   URL: POST /admin/purchases/{id}/receive
   FUNGSI: **TERIMA BARANG → STOK MASUK**
   ACTION: Call PurchaseService::receivePurchase()
   RESULT: Stok bertambah + mutasi tercatat
   REDIRECT: admin.purchases.show
   FLASH: "Pembelian berhasil diterima. Stok telah ditambahkan."

8. cancel(Request $request, Purchase $purchase)
   URL: POST /admin/purchases/{id}/cancel
   FUNGSI: Batalkan purchase
   INPUT: reason (required)
   REDIRECT: admin.purchases.show
   FLASH: success message

9. destroy(Purchase $purchase)
   URL: DELETE /admin/purchases/{id}
   FUNGSI: Hapus purchase (HANYA DRAFT)
   VALIDATION: Exception jika bukan draft
   REDIRECT: admin.purchases.index
   FLASH: success message
