## 1. TARGET TUGAS KETIGA

1. Menyambungkan form di layar POS (`pos/sales/create`) ke backend:
   - User pilih produk + qty → dikirim ke server.
   - Setelah submit, data tersimpan ke DB.

2. Membuat alur transaksi POS:
   - Buat 1 route POST untuk menyimpan transaksi.
   - Di controller, gunakan **Form Request** untuk validasi.
   - Simpan:
     - `sales` (header transaksi)
     - `sale_items` (produk per baris)
     - `payments` (informasi pembayaran)
   - Hitung total, kembalian, dll.

3. Mutasi Stok:
   - Untuk setiap item terjual → kurangi stok di `stocks`.
   - Tambahkan record di `stock_mutations` dengan tipe `out`.

4. Membuat **Service Layer**:
   - `SaleService` untuk handle transaksi penjualan.
   - `StockService` untuk handle mutasi stok.

> Jangan ubah struktur tabel yang sudah ada, kecuali memang perlu menambah kolom ringan (misal `sale_number` atau `session_id`), dan jelaskan dulu.

## 2. RUTE & ALUR TRANSAKSI POS

### 2.1. Route

Di `routes/web.php` (dalam group `pos`), tambahkan:

- `POST /pos/sales` → `POS\SaleController@store`, nama route: `pos.sales.store`.

Contoh:

```php
Route::prefix('pos')
    ->name('pos.')
    ->group(function () {
        Route::get('/dashboard', [\App\Http\Controllers\POS\DashboardController::class, 'index'])->name('dashboard');
        Route::get('/sales', [\App\Http\Controllers\POS\SaleController::class, 'create'])->name('sales.create');
        Route::post('/sales', [\App\Http\Controllers\POS\SaleController::class, 'store'])->name('sales.store');
    });
