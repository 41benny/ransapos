3. SERVICE LAYER ✅
================================================================================

File: app/Services/CashSessionService.php

METHODS:
--------

✅ openSession($data, $user)
   - Generate session number unik
   - Format: CS-{outlet_code}-{date}-{sequence}
   - Example: CS-OUT001-20251107-001
   - Buat record cash_session baru
   - Status: open
   - Expected balance = opening balance
   - Transaction handling

✅ closeSession($session, $actualBalance, $notes)
   - Validasi session masih open
   - Hitung selisih (actual - expected)
   - Update status: closed
   - Set closed_at timestamp
   - Append notes
   - Transaction handling

✅ getActiveSessionFor($user, $outletId)
   - Ambil session aktif untuk user & outlet
   - Filter: status = open
   - Order by opened_at desc
   - Return dengan relasi (outlet, user)

✅ generateSessionNumber($outletId)
   - Format: CS-{outlet_code}-{date}-{seq}
   - Auto-increment per outlet per hari
   - 3 digit sequence (001-999)

✅ getSessionHistory($filters)
   - Filter by: outlet, user, status, date range
   - Pagination (15 per page)
   - With relasi: outlet, user
   - Order by opened_at desc

Features:
✅ DB Transaction handling
✅ Exception handling
✅ Eager loading relasi
✅ Auto-increment sequence
✅ Flexible filtering

6. ALUR LENGKAP
================================================================================

OPEN SHIFT (Buka Shift):
-------------------------
1. Kasir buka POS Dashboard
2. Jika tidak ada session → Lihat tombol "Buka Shift Kasir"
3. Klik tombol → Redirect ke /pos/sessions/open
4. Pilih outlet
5. Isi saldo awal (default: 500.000)
6. Isi notes (optional)
7. Klik "Buka Shift"
8. Validasi:
   - Outlet valid?
   - Saldo awal >= 0?
   - Sudah ada session open?
9. Jika valid:
   - Generate session number: CS-OUT001-20251107-001
   - Simpan ke cash_sessions
   - Set status = open
   - Set opened_at = now
10. Redirect ke dashboard dengan success message
11. Dashboard menampilkan info session aktif

TRANSAKSI (Dengan Session):
----------------------------
1. Session aktif → Bisa akses /pos/sales
2. Pilih produk → Add to cart
3. Submit transaksi
4. SaleService->createSale():
   - Simpan sales, sale_items, payments
   - Update cash_sessions:
     * total_sales += amount
     * total_cash atau total_non_cash += amount
     * expected_balance = opening + total_cash
5. Success → Cart reset

CLOSE SHIFT (Tutup Shift):
---------------------------
1. Kasir klik "Tutup Shift" di dashboard
2. Redirect ke /pos/sessions/close
3. Tampilkan:
   - Session info
   - Sales summary
   - Expected balance calculation
   - Recent transactions
4. Kasir hitung uang fisik di laci
5. Input actual balance
6. JavaScript real-time:
   - Calculate: difference = actual - expected
   - Show preview dengan warna
   - Update text: "+ Rp X (Lebih)" atau "- Rp X (Kurang)"
7. Isi notes (optional)
8. Klik "Tutup Shift"
9. Konfirmasi: "Apakah Anda yakin?"
10. Submit:
    - Update cash_sessions:
      * actual_balance = input
      * difference = calculated
      * status = closed
      * closed_at = now
11. Redirect ke dashboard dengan message:
    - "Shift berhasil ditutup! (Lebih Rp X)" atau
    - "Shift berhasil ditutup! (Kurang Rp X)"

VIEW HISTORY (Admin):
---------------------
1. Admin buka sidebar → "Shift Kasir"
2. Redirect ke /admin/cash-sessions
3. Lihat list semua sessions
4. Filter:
   - Pilih outlet
   - Pilih kasir
   - Pilih status (open/closed)
   - Pilih date range
   - Klik "Filter"
5. Table update dengan hasil filter
6. Lihat detail:
   - Session number
   - Outlet & kasir
   - Opened/closed time
   - Sales & balance info
   - Selisih (color coded)
7. Pagination untuk navigate
