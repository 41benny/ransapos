3. PURCHASE SERVICE (BUSINESS LOGIC LAYER)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

File: app/Services/PurchaseService.php
Dependency Injection: StockService

┌──────────────────────────────────────────────────────────────────────────┐
│                         PUBLIC METHODS                                    │
└──────────────────────────────────────────────────────────────────────────┘

1. createPurchase(array $data): Purchase
   ─────────────────────────────────────────────────────────────
   Fungsi: Membuat purchase baru dengan status 'draft'
   
   Input Data:
     - outlet_id          (required)
     - supplier_id        (required)
     - purchase_date      (required)
     - tax_amount         (optional, default: 0)
     - discount_amount    (optional, default: 0)
     - payment_status     (optional, default: 'pending')
     - notes              (optional)
     - items[]            (required, min: 1)
       - product_id       (required)
       - quantity         (required)
       - unit_price       (required)
       - discount_amount  (optional, default: 0)
   
   Proses:
     [1] Generate purchase_number otomatis
     [2] Hitung subtotal dari semua items
     [3] Hitung total = subtotal + tax - discount
     [4] Simpan purchase header
     [5] Simpan purchase items
     [6] Return purchase dengan relasi
   
   Output:
     Purchase object dengan relasi:
     - items.product
     - outlet
     - supplier
     - creator
   
   Status Hasil: 'draft'
   Efek ke Stok: ❌ TIDAK ADA (stok belum bertambah)

   ─────────────────────────────────────────────────────────────

2. updatePurchase(Purchase $purchase, array $data): Purchase
   ─────────────────────────────────────────────────────────────
   Fungsi: Update purchase yang masih draft
   
   Validasi: Hanya bisa update jika status = 'draft'
   
   Proses:
     [1] Validasi status = 'draft'
     [2] Hitung ulang subtotal dan total
     [3] Update purchase header
     [4] Hapus items lama
     [5] Buat items baru
     [6] Return updated purchase
   
   Efek ke Stok: ❌ TIDAK ADA (masih draft)

   ─────────────────────────────────────────────────────────────

3. receivePurchase(Purchase $purchase): Purchase ⭐ KEY METHOD
   ─────────────────────────────────────────────────────────────
   Fungsi: Terima barang & TAMBAH STOK OTOMATIS
   
   Validasi: Hanya bisa receive jika status = 'draft'
   
   Proses:
     [1] Validasi status = 'draft'
     [2] Update purchase:
         - status = 'received'
         - received_at = now()
         - received_by = auth()->id()
     [3] Loop setiap item:
         → Call StockService::addPurchaseStock()
           ┌────────────────────────────────────────┐
           │  ✅ STOK BERTAMBAH DI TABEL STOCKS    │
           │  ✅ MUTASI TERCATAT DI STOCK_MUTATIONS │
           └────────────────────────────────────────┘
     [4] Commit transaction
     [5] Return purchase dengan relasi lengkap
   
   Efek ke Stok: ✅ STOK BERTAMBAH SESUAI QTY ITEMS
   Mutasi: ✅ TERCATAT (type: 'in', reference: 'purchase')
   
   ⚠️ PENTING: Setelah received, purchase TIDAK BISA diubah/dihapus!

   ─────────────────────────────────────────────────────────────

4. cancelPurchase(Purchase $purchase, string $reason): Purchase
   ─────────────────────────────────────────────────────────────
   Fungsi: Batalkan purchase
   
   Validasi: Tidak bisa cancel jika sudah received
   
   Proses:
     [1] Validasi status != 'received'
     [2] Update:
         - status = 'cancelled'
         - notes = append reason
     [3] Return purchase
   
   Efek ke Stok: ❌ TIDAK ADA (stok tidak berubah)

   ─────────────────────────────────────────────────────────────

5. generatePurchaseNumber(int $outletId): string
   ─────────────────────────────────────────────────────────────
   Fungsi: Generate nomor PO unik
   
   Format: PO-{outlet}-{date}-{sequence}
   
   Contoh:
     - PO-001-20251107-001
     - PO-001-20251107-002
     - PO-002-20251107-001
   
   Logic:
     - Ambil last purchase hari ini untuk outlet ini
     - Extract sequence terakhir
     - Increment + 1
     - Pad dengan leading zero (3 digit)

   ─────────────────────────────────────────────────────────────

6. getPurchases(array $filters): LengthAwarePaginator
   ─────────────────────────────────────────────────────────────
   Fungsi: Ambil list purchases dengan filter
   
   Filter Options:
     - outlet_id      (filter by outlet)
     - supplier_id    (filter by supplier)
     - status         (filter by status)
     - date_from      (filter tanggal >= date_from)
     - date_to        (filter tanggal <= date_to)
   
   Eager Load:
     - outlet
     - supplier
     - creator
     - items
   
   Order: purchase_date DESC, id DESC
   Pagination: 15 per page

4. STOCK SERVICE INTEGRATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

File: app/Services/StockService.php

Method yang Digunakan: addPurchaseStock()

Signature:
  addPurchaseStock(
      int $productId,
      int $outletId,
      float $quantity,
      int $purchaseId,
      ?int $userId = null
  ): void

Proses:
  [1] Ambil/buat record di tabel 'stocks'
      WHERE product_id = $productId AND outlet_id = $outletId
  
  [2] Simpan stock_before = current quantity
  
  [3] Tambah stok:
      stocks.quantity += $quantity
      stocks.last_mutation_at = now()
  
  [4] Catat mutasi di 'stock_mutations':
      - product_id
      - outlet_id
      - mutation_type: 'in'
      - quantity: +$quantity (positif)
      - stock_before
      - stock_after
      - reference_type: 'purchase'
      - reference_id: $purchaseId
      - mutation_date: today
      - notes: 'Pembelian'
      - created_by: $userId
  
  [5] Commit transaction

Flow Integration:
  PurchaseService::receivePurchase()
    └─→ Loop items
        └─→ StockService::addPurchaseStock()
            └─→ Update stocks ✅
            └─→ Insert stock_mutations ✅

11. DATA INTEGRATION MATRIX
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Tabel yang Terlibat:
  ┌────────────────────┬───────────────────────────────────────────────┐
  │ Tabel              │ Operasi                                       │
  ├────────────────────┼───────────────────────────────────────────────┤
  │ purchases          │ INSERT, UPDATE, DELETE                        │
  │ purchase_items     │ INSERT, DELETE (cascade)                      │
  │ stocks             │ UPDATE (saat receive)                         │
  │ stock_mutations    │ INSERT (saat receive)                         │
  │ outlets            │ READ (relasi)                                 │
  │ suppliers          │ READ (relasi)                                 │
  │ products           │ READ (relasi)                                 │
  │ users              │ READ (creator, receiver)                      │
  └────────────────────┴───────────────────────────────────────────────┘

Relasi Foreign Keys:
  purchases.outlet_id     → outlets.id (RESTRICT)
  purchases.supplier_id   → suppliers.id (RESTRICT)
  purchases.created_by    → users.id (SET NULL)
  purchases.received_by   → users.id (SET NULL)
  
  purchase_items.purchase_id  → purchases.id (CASCADE)
  purchase_items.product_id   → products.id (RESTRICT)
  
  stock_mutations.product_id  → products.id
  stock_mutations.outlet_id   → outlets.id
  stock_mutations.created_by  → users.id

Cascade Rules:
  ✅ Delete purchase → auto delete items (CASCADE)
  ❌ Cannot delete outlet if ada purchases (RESTRICT)
  ❌ Cannot delete supplier if ada purchases (RESTRICT)
  ❌ Cannot delete product if ada purchase_items (RESTRICT)
