1. COGS (COST OF GOODS SOLD) PER ITEM TRACKING
================================================================================

A. Database Enhancement:
   File: database/migrations/2025_11_12_130000_add_cogs_to_sale_items_table.php
   
   Tambahan kolom di sale_items:
   - cogs DECIMAL(15,2) DEFAULT 0 → Cost of Goods Sold per item

B. Model Update:
   File: app/Models/SaleItem.php
   
   Tambahan:
   - Fillable: 'cogs'
   - Cast: 'cogs' => 'decimal:2'

C. Logika Perhitungan COGS:
   File: app/Services/SaleService.php
   
   Method baru: calculateItemCogs()
   
   Logika per product_type:
   ┌─────────────────────────────────────────────────────────────────┐
   │ 1. service → COGS = 0 (tidak ada biaya material)               │
   │                                                                 │
   │ 2. raw_material → COGS = purchase_price × quantity             │
   │                                                                 │
   │ 3. finished_good:                                               │
   │    • Jika ada BOM aktif:                                        │
   │      COGS = Σ(component.purchase_price × detail.quantity       │
   │                × sale_quantity)                                 │
   │    • Jika tidak ada BOM:                                        │
   │      COGS = purchase_price × quantity (fallback)               │
   └─────────────────────────────────────────────────────────────────┘

D. Benefits:
   - Profit analysis per menu item
   - Akurat untuk produk dengan BOM (real material cost)
   - Support untuk laporan margin keuntungan detail

3. KONFIGURASI NEGATIVE STOCK (OPTIONAL)
================================================================================

A. Environment Configuration:
   File: .env.example
   
   Tambahan:
   # Stock Management
   ALLOW_NEGATIVE_STOCK=false

B. App Configuration:
   File: config/app.php
   
   Tambahan:
   'allow_negative_stock' => env('ALLOW_NEGATIVE_STOCK', false),

C. Logic Implementation:
   Files: 
   - app/Services/StockService.php → reduceSaleStock()
   - app/Services/SaleService.php → BOM component validation
   
   Behavior:
   ┌─────────────────────────────────────────────────────────────────┐
   │ ALLOW_NEGATIVE_STOCK = false (default):                        │
   │ → Tolak transaksi jika stok < kebutuhan                        │
   │ → Throw Exception dengan pesan jelas                           │
   │                                                                 │
   │ ALLOW_NEGATIVE_STOCK = true:                                    │
   │ → Izinkan stok minus                                            │
   │ → Transaksi tetap jalan, stok bisa negatif                     │
   │ → Berguna untuk pre-order atau back-order scenarios            │
   └─────────────────────────────────────────────────────────────────┘

D. Use Cases:
   - Restaurant dengan pre-order system
   - Inventory yang allow back-order
   - Testing environments
   - Emergency sales situations
