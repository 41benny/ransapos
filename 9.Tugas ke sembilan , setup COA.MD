## TUGAS KESEMBILAN – COA (AKUN BIAYA/PENDAPATAN) + HPP + LAPORAN LABA RUGI

---

## 0. KONTEKS (SETELAH TUGAS 1–8)

Sampai sekarang sistem sudah punya:

1. ✅ POS + stok + mutasi:
   - `sales`, `sale_items`, `stock_mutations` (IN/OUT)
   - POS sudah live, stok berkurang otomatis saat penjualan.

2. ✅ Pembelian & restock:
   - `purchases`, `purchase_items`
   - Receive purchase → stok bertambah, mutasi IN tercatat.

3. ✅ Kas & Bank:
   - `cash_accounts` (kas/bank master) + `cash_transactions` (kas masuk/keluar)
   - Pembayaran purchase → kas keluar, `purchase.payment_status` update.

4. ✅ Laporan operasional lain:
   - Laporan penjualan, shift kasir, mutasi kas.

Tapi:
- Belum ada **COA (Chart of Accounts)** untuk klasifikasi biaya & pendapatan.
- Belum ada **HPP** yang jelas (nilai cost dari stok keluar).
- Belum ada **Laporan Laba Rugi** (Pendapatan – HPP – Biaya = Laba Bersih).

Tugas ini fokus:
- Menambahkan **master akun pendapatan/biaya (COA)**.
- Menghubungkan **kas keluar** dengan akun biaya.
- Menambahkan **cost/HPP** pada stok keluar.
- Membuat **Laporan Laba Rugi sederhana**.

Gunakan Bahasa Indonesia untuk komentar & penjelasan.

---

## 1. TUJUAN TUGAS KESEMBILAN

1. Membuat tabel & modul **COA (Chart of Accounts)**:
   - Minimal tipe `income` dan `expense`.
   - Bisa dikelompokkan (Pendapatan, HPP, Biaya Operasional).

2. Menghubungkan **cash_transactions** dengan akun COA:
   - Transaksi kas keluar wajib pilih akun biaya (expense).
   - Transaksi kas masuk opsional pilih akun pendapatan (income).

3. Menambahkan **nilai HPP** berdasarkan stok keluar:
   - Setiap `stock_mutations` type `out` terkait penjualan punya:
     - `unit_cost`
     - `total_cost`
   - Sumber cost: field `cost_price` di `products` (atau cost default yang ada).

4. Membuat **Laporan Laba Rugi**:

   Untuk periode tertentu (dan bisa filter outlet):

   - Pendapatan: dari `sales` (grand_total).
   - HPP: dari `stock_mutations` OUT (total_cost).
   - Biaya Operasional: dari `cash_transactions` yang COA-nya type `expense`.
   - Hitung:
     - Laba Kotor  = Pendapatan – HPP
     - Laba Bersih = Laba Kotor – Total Biaya Operasional

## 2. STRUKTUR DATABASE – COA & HPP

### 2.1. Tabel coa_accounts

Buat migration baru:

- Nama tabel: `coa_accounts`
- Model: `App\Models\CoaAccount`

Kolom minimal:

- `id`
- `code` (string, unique) → seperti `4-100`, `5-200`, dsb.
- `name` (string) → mis: “Penjualan Resto”, “Biaya Listrik”, “Biaya Administrasi & Umum”.
- `type` (enum/string) → `income`, `expense` (kalau mau bisa tambah `asset`, `liability`, `equity` tapi minimal 2 dulu).
- `group` (string) → mis: `PENDAPATAN`, `HPP`, `BIAYA OPERASIONAL`.
- `is_active` (boolean, default true).
- `notes` (nullable text).
- timestamps.

Relasi dasar:

```php
class CoaAccount extends Model
{
    protected $fillable = [
        'code',
        'name',
        'type',
        'group',
        'is_active',
        'notes',
    ];

    public function cashTransactions()
    {
        return $this->hasMany(CashTransaction::class, 'coa_account_id');
    }
}
